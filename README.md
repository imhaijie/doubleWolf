# 🌙 双身份狼人杀面杀助手

一个专为线下狼人杀面杀设计的实时辅助工具，支持**双身份**玩法，手机打开即用。

## ✨ 特性

### 🎮 核心玩法
- **双身份系统**：每个玩家拥有两个身份，第一身份死亡后第二身份生效
- **阵营判定**：狼人阵营永久归属，不随死亡改变
- **胜利条件**：
  - 好人胜利：所有狼人阵营玩家双身份全灭
  - 狼人胜利：所有"金水宝宝"死亡 或 所有神职玩家死亡

### 🌙 夜晚阶段（全自动流程）
固定顺序：**狼人 → 预言家 → 女巫 → 守卫 → 猎人触发环节**

1. **狼人行动**：存活狼人独立选择目标，系统取票高者
2. **预言家行动**：选择查验目标，系统显示结果（仅自己可见）；结果会在界面上停留至少 2 秒
3. **女巫行动**：分步提示"用解药？"→"用毒药？"（首夜可自救）
4. **守卫行动**：选择守护目标，校验"不可连守"规则
5. **系统结算**：综合刀/药/守计算实际死亡
6. **猎人触发**：若猎人夜晚死亡且非毒杀，弹出开枪选择

> 每个夜晚子阶段会按照固定顺序依次展开，即便当前没有符合条件的玩家也会经过该环节，系统会在 20 秒后自动跳过并通过声音/文字通知房主。
### ☀️ 白天阶段（全自动流程）

#### 📢 死亡公告
- 顶部居中温和提示（3 秒自动淡出）
- 严格隐藏身份类型、死亡原因

#### ⏱️ 发言计时
- 房主赛前设置：总发言时长（5/10/15 分钟/无限）
- 顶部全局倒计时条（红→黄→绿渐变警示）
- 倒计时归零自动进入投票阶段，或房主可点击“提前结束发言”按钮立即进入下一环节

#### 💥 白狼王自爆
- 发言阶段，当前身份为白狼王的玩家界面常驻闪烁"自爆"按钮
- 点击后选择 1 名玩家，双人立即出局
- 跳过投票环节，直接检测胜利条件，进入下一夜晚

#### 🗳️ 投票环节
- 玩家勾选 1 名目标提交（提交后锁定为"✓ 已投票"）
- 他人界面仅显示"● 已投票"（完全匿名）
- 实时计票进度条动态更新
- 得票最高者出局，平票则重新投票，平票者不能投票，由其他玩家再次投票，并且只能在平票者中投
- 猎人触发：若出局者为猎人且非毒杀，立即弹出开枪选择

### 🔔 房主通知机制
- 每组完成后（含猎人环节），房主设备自动播放 1 秒提示音 + 通知文字
- 仅夜晚阶段通知，绝不透露技能细节
- 白天阶段房主与普通玩家体验完全一致

### 🌐 全链路实时同步
- Socket.IO 双向通信，零刷新体验
- 断线重连：Cookie+LocalStorage 保障恢复对局
- 24 小时无活动房间自动清理

## 🚀 快速开始

### 本地开发

```bash
# 克隆项目
git clone https://github.com/xxx/werewolf-helper.git
cd werewolf-helper

# 安装依赖
npm install

```

```bash

npm run build
```

### 跨设备访问
- 手机/平板连接同一 WiFi
- 输入电脑 IP:3000 即可加入房间
- 支持自定义房间号（6 位字母数字）

## 📋 角色配置

### 默认角色池（9 人标准局）
| 角色 | 数量 | 阵营 | 是否神职 |
|------|------|------|----------|
| 狼人 | 2 | 狼人 | 否 |
| 白狼王 | 1 | 狼人 | 是 |
| 预言家 | 1 | 好人 | 是 |
| 女巫 | 1 | 好人 | 是 |
| 猎人 | 1 | 好人 | 是 |
| 守卫 | 1 | 好人 | 是 |
| 平民 | 2 | 好人 | 否 |

### 预设模板
- **9 人标准局**：3 狼 +3 神 +3 民
- **12 人标准局**：4 狼 +4 神 +4 民
- **自定义配置**：房主可自由调整各角色数量

## 🎯 角色技能

| 角色 | 技能说明 |
|------|----------|
| 🐺 狼人 | 夜晚选择击杀一名玩家（独立选择→系统取票高者） |
| 👑 白狼王 | 白天发言阶段可自爆，带走一名玩家 |
| 🔮 预言家 | 夜晚查验一名玩家，得知其阵营（查杀/金水） |
| 🧙‍♀️ 女巫 | 夜晚分步使用解药和毒药（首夜可自救） |
| 🏹 猎人 | 死亡时（非毒杀）可开枪带走一名玩家 |
| 🛡️ 守卫 | 夜晚守护一名玩家（不可连续守护同一人） |
| 👤 平民 | 无特殊技能，白天投票放逐狼人 |

## 📱 使用说明

### 创建房间
1. 输入你的名字
2. 选择游戏设置（发言时间、投票时间）
3. 选择角色配置（预设或自定义）
4. 点击"创建房间"
5. 分享房间码给其他玩家

### 加入房间
1. 输入你的名字
2. 输入 6 位房间码
3. 点击"加入房间"

### 游戏流程
1. **等待阶段**：所有玩家选择双身份
2. **开始游戏**：房主点击开始
3. **夜晚阶段**：各角色按顺序行动
4. **白天阶段**：死亡公告→发言→投票
5. **游戏结束**：显示复盘和完整身份

## 🛠️ 技术栈

- **前端**：Next.js 16 + React 19 + TypeScript
- **后端**：Node.js + Express + Socket.IO
- **UI 组件**：shadcn/ui + Tailwind CSS
- **动画**：Framer Motion
- **状态管理**：React Hooks + LocalStorage

## 🔒 隐私保护

- **身份绝对隐藏**：游戏进行中仅自己可见身份
- **死亡信息脱敏**：仅显示"阵亡"，不透露身份和死因
- **复盘可见**：游戏结束后全员可见完整身份和操作时间轴

## 🎨 设计原则

1. **房主是平等玩家**：仅赛前设置、踢人、开始/中止权限
2. **面杀沉浸感**：无聊天框/无语音/无弹幕干扰
3. **新手零门槛**：角色选择页带技能说明
4. **技术隐形，规则显形**：复杂逻辑静默运行

## 📝 开发说明

> **开发规范**：所有改动在提交到仓库前必须先在本地运行 `npm run build`（或 `npm run dev`）确认没有编译/运行错误。推荐添加 pre-push 钩子自动执行构建。


### 添加新角色
1. 在 `server/roles.ts` 中定义角色
2. 在 `server/types.ts` 中添加类型
3. 在 `lib/types.ts` 中添加显示信息
4. 在 `server/game-manager.ts` 中实现技能逻辑

### 修改胜利条件
编辑 `server/victory-check.ts` 中的 `checkVictory` 函数。

### 自定义通知音
替换 `public/notification.mp3` 文件。
